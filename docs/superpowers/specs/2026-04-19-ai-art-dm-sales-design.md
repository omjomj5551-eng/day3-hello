# AI 이미지 작품 판매 정적 웹사이트 디자인 스펙

- 작성일: 2026-04-19
- 범위: MVP (원페이지 정적 사이트)
- 목표: 인스타그램 DM 문의 전환 극대화

## 1) 프로젝트 목표
AI 이미지 작품(디지털 파일)을 소개하고, 방문자가 빠르게 인스타그램 DM으로 문의하도록 유도한다.

### 성공 기준
- 모든 핵심 구간에서 DM CTA 노출
- 첫 화면에서 1회 스크롤 내 작품/문의 가치 전달
- 모바일에서 CTA 접근성 우선

## 2) 범위와 비범위
### 범위
- 원페이지 구성
- 대표 작품 갤러리(6~9개)
- 문의 유도 CTA(인스타그램 DM)
- 라이선스 개별 협의 안내
- FAQ 섹션

### 비범위
- 자체 결제/장바구니/주문관리
- 회원가입/로그인
- 다국어/블로그/관리자 페이지

## 3) 정보 구조(IA)
1. 히어로 섹션
2. 대표 작품 그리드
3. 구매/라이선스 안내
4. FAQ
5. 하단 CTA

## 4) UX 설계
- CTA 문구 통일: "작품 문의하기 (Instagram DM)"
- CTA 배치: 상단/중단/하단 반복
- 모바일 우선 레이아웃 적용
- 문의 템플릿 제공:
  - 관심 작품명
  - 사용 목적
  - 원하는 해상도

## 5) 콘텐츠/카피 가이드
- 톤: 프리미엄, 간결
- 히어로 예시 카피: "감성적 AI 아트, 당신의 공간과 브랜드를 위한 디지털 작품"
- 라이선스 고지: "모든 작품은 사용 목적에 따라 라이선스를 개별 협의합니다."
- 전달 가이드: 협의 후 24~72시간 내 전달(상황에 따라 조정)

## 6) 컴포넌트 설계
- Hero
  - 헤드라인, 서브카피, 메인 CTA
- ArtworkGrid
  - 썸네일 카드 6~9개
  - 카드 내 제목/스타일 태그(선택)
- PolicyBlock
  - 디지털 파일 제공 범위
  - 라이선스 개별 협의 문구
- FAQAccordion
  - 상업 사용 가능 여부
  - 수정 요청 가능 범위
  - 환불/취소 기준
- FooterCTA
  - 반복 CTA + 인스타 계정 링크

## 7) 데이터 흐름
정적 데이터(작품 목록, FAQ, 정책 문구)를 페이지 렌더링에 사용한다.
사용자 상호작용의 핵심 이벤트는 CTA 클릭이며, 외부 인스타그램 프로필/DM 진입으로 흐름을 종료한다.

## 8) 성능/접근성
- 이미지: WebP 우선, 적절한 압축
- 갤러리 이미지 지연 로딩 적용
- 버튼 대비/터치 영역 확보
- 주요 섹션의 명확한 제목 계층 유지

## 9) 오류/예외 처리
정적 사이트 특성상 런타임 오류 가능성은 낮다. 핵심 예외는 외부 링크 문제다.
- 인스타 링크 실패 대비: 동일 링크를 2개 이상 위치에 중복 제공
- 계정 변경 가능성 대비: 링크 값을 단일 설정 지점에서 관리

## 10) 테스트 전략
- 수동 확인(필수)
  - 모바일/데스크톱에서 레이아웃 확인
  - 모든 CTA 클릭 시 올바른 인스타 링크 이동 확인
  - 이미지 로딩 속도 및 깨짐 여부 확인
- 최소 기술 검증
  - HTML/CSS 유효성 기본 점검
  - 콘솔 에러 없음 확인

## 11) 릴리스 기준
- 원페이지 섹션 구현 완료
- CTA 동작 100% 확인
- 핵심 카피/FAQ/정책 문구 반영 완료
- 모바일 시인성/탭 가능 영역 확인 완료

## 12) 확장 방향(후속)
- 작품 상세 템플릿 1종 추가
- 작가 소개 섹션 보강
- 문의 추적용 링크 파라미터(캠페인 단위) 추가
